# Proyecto_DGI

## Descripción del Proyecto

### Problema
Muchas PyMEs realizan el control de inventarios mediante hojas de cálculo o registros físicos manuales. Esto genera desajustes en el stock, pérdida de datos históricos, lentitud para localizar mercancía y retrasos al generar reportes para la toma de decisiones.

### Objetivo General
Desarrollar una aplicación web para la gestión de inventarios que centralice la administración de productos, controle los flujos de entradas y salidas de mercancía, monitoree el stock en tiempo real y genere reportes analíticos básicos.

### Alcance
* Registro, edición y baja de productos.
* Consulta de existencias y buscador por nombre o categoría.
* Registro de movimientos (entradas/salidas de almacén).
* Alertas visuales de stock bajo.
* Gestión de usuarios y niveles de acceso.

### Fuera de Alcance
* Facturación electrónica (SAT).
* Módulo de compras/e-commerce.
* Sincronización con ERPs externos.
* App móvil nativa.


## Tecnologías Utilizadas
* **Frontend:** React, HTML5, CSS3, JavaScript
* **Backend:** Node.js, Express
* **Base de Datos:** MySQL
* **IDE:** Visual Studio Code
* **Gestión & DevOps:** Jira, GitHub, Vercel, Railway
